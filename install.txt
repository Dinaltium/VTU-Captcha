Install instructions for this project

Quick notes:
- This project was developed on Ubuntu (WSL) and expects Python 3.8+ and Node.js (for the frontend).
- Recommended: use WSL (Ubuntu 22.04) on Windows for best compatibility. Where commands differ, both WSL/Linux and Windows instructions are provided.
- The repo contains a `requirements.txt` (root) — prefer `pip install -r requirements.txt` first.

-----------------------------
1) WSL / Ubuntu 22.04 (recommended)
-----------------------------

# 1. System packages (copy-paste)
sudo apt update
sudo apt install -y python3 python3-venv python3-pip git curl build-essential unzip libglib2.0-0 libnss3 libx11-6 libxss1 libxcomposite1 libasound2 fonts-liberation

# 2. (Optional) Google Chrome (if you want a stable Chrome binary for chromedriver)
wget -q -O /tmp/google-chrome.deb https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install -y /tmp/google-chrome.deb || sudo apt --fix-broken install -y

# 3. Create and activate Python venv
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip

# 4. Install Python requirements (preferred)
pip install -r requirements.txt

# 5. Extra/known packages (copy-paste if some packages fail or are missing)
pip install webdriver-manager==4.0.2
pip install selenium
# If you don't need GPU acceleration, install CPU-only TensorFlow (avoids CUDA setup):
pip install "tensorflow-cpu"
# If you need GPU TensorFlow, see the CUDA section below.

# 6. Chromedriver handling
# - There's a `backend/chromedriver` in the repo. If it doesn't match your Chrome, use webdriver-manager to auto-download:
python -c "from webdriver_manager.chrome import ChromeDriverManager; print(ChromeDriverManager().install())"

# 7. Frontend (Node.js + dependencies)
# install Node.js 18 (LTS) on Ubuntu:
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

cd frontend
npm install
# to run dev server:
npm run dev
# to build static files:
npm run build
cd ..

# 8. Run the backend (example)
# Activate the venv if not active: `source .venv/bin/activate`
python -m python.api || python python/api.py || python backend/api.py
# Note: adjust the actual entrypoint if needed (see `backend/` and `python/` folders).

-----------------------------
2) Windows (native) — cmd.exe / PowerShell
-----------------------------

# 1. Install Python
# - Download from https://www.python.org/downloads/ and check "Add Python to PATH".
# 2. Open a new terminal (cmd.exe or PowerShell) and run:
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
pip install webdriver-manager==4.0.2
pip install selenium

# 3. Install Google Chrome from https://www.google.com/chrome/
# 4. Install chromedriver (matching Chrome version):
# - Easiest: use webdriver-manager (see above) from Python.

# 5. Node.js (Windows):
# - Download and install Node.js 18 LTS from https://nodejs.org/
cd frontend
npm install
npm run dev

-----------------------------
Optional: GPU / CUDA for TensorFlow (advanced)
-----------------------------

- GPU TensorFlow requires matching CUDA + cuDNN versions for the TensorFlow release you use.
- If you want GPU acceleration, do NOT install `tensorflow-cpu`; instead check TensorFlow's install guide:
  https://www.tensorflow.org/install/source#gpu

Common compatibility (example only, verify before installing):
- TensorFlow 2.13 ~ CUDA 11.8 and cuDNN 8.6

Basic steps (Ubuntu) — verify exact versions on TF site:
1) Install NVIDIA driver (use your distro's instructions or https://developer.nvidia.com/cuda-downloads)
2) Install CUDA toolkit (matching TF)
3) Install cuDNN (matching CUDA)
4) Reboot and verify `nvidia-smi` works
5) pip install tensorflow (GPU-enabled wheel)

If you cannot or do not want to manage CUDA, use CPU-only with `pip install tensorflow-cpu`.

-----------------------------
Other helpful notes
-----------------------------
- If `pip install -r requirements.txt` fails, try installing problematic packages individually and inspect error messages.
- If chromedriver file has an alternate data stream (Windows `:Zone.Identifier`), you can remove it by copying/chmod or re-downloading the chromedriver under WSL/Ubuntu.
- To check Chrome version (Linux): `google-chrome --version` or `chromium-browser --version`.
- To check chromedriver version: `./chromedriver --version` (make sure executable permissions: `chmod +x backend/chromedriver`).
